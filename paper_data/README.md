# AnonShield Paper Data

Benchmark data, datasets, and results for the AnonShield paper evaluating
privacy-preserving anonymization of vulnerability scan reports across multiple
file formats and tool versions.

---

Each folder inside `results/` contains:
- `benchmark_results.csv` — performance measurements (wall-clock, CPU, memory, GPU, throughput)
- `run_logs/` — per-file, per-strategy execution logs

## Benchmark Results

Each `benchmark_results.csv` contains one row per (file × version × strategy × run_number) combination and the following columns:

| Column | Description |
|---|---|
| `version` | AnonShield version (1.0 / 2.0 / 3.0) |
| `strategy` | Anonymization strategy (default / filtered / hybrid / standalone / presidio) |
| `file_name` | Input file name |
| `file_path` | Absolute input file path at time of run |
| `file_extension` | Extension (.csv, .json, .xml, …) |
| `run_number` | Run index (1–N) |
| `timestamp` | ISO-8601 timestamp of run start |
| `measurement_mode` | `single_file` or `overhead_calibration` |
| `status` | `SUCCESS` or `FAILED` |
| `error_message` | Error text if status = FAILED (empty otherwise) |
| `wall_clock_time_sec` | Elapsed real time (seconds) — **primary metric** |
| `user_time_sec` | CPU user time |
| `system_time_sec` | CPU system time |
| `cpu_percent` | CPU utilization % during run |
| `max_resident_set_kb` | Peak RSS memory (KB) |
| `throughput_kb_per_sec` | Input KB processed per wall-clock second |
| `gpu_available` | Whether a GPU was detected |
| `avg_gpu_utilization_percent` | Mean GPU utilization % |
| `peak_gpu_memory_used_mb` | Peak GPU memory consumption (MB) |

## Anonymization Config Effect

The `anonymization_config.json` / `anonymization_config_cve.json` pre-builds an
entity recognition cache of up to 200,000 entities extracted from the target
dataset before processing begins. This trades a one-time upfront cost for
dramatically reduced per-run execution times:

| Dataset | Without config | With config | Speedup |
|---|---|---|---|
| D2 CSV | ~1780 s/run | ~13 s/run | **~135×** |
| D2 JSON | ~880 s/run | ~19 s/run | **~47×** |
| D3 CSV | ~240 s/run | ~9 s/run | **~28×** |
| D3 JSON | ~880 s/run | ~21 s/run | **~42×** |

## How to Reproduce

This section is written for **artifact evaluators** who want to verify the
benchmark results from the paper. Read it fully before running anything.

---

### Script Overview

| Script | Purpose | Runtime |
|---|---|---|
| `test_minimal/run_tests.sh` | **Start here.** Smoke test with tiny datasets to verify the full pipeline works end-to-end in ~5–20 min. Runs D1, D1C, D2, D3 (500-row subsets), all versions, all strategies, then generates analysis. | ~5–20 min |
| `scripts/reproduce_all_runs.sh` | Full benchmark reproduction using the complete original datasets. Produces results that match the paper's tables exactly. | ~60–80 h |
| `scripts/analyze_all.sh` | Generates charts, statistics, and PDFs from existing `benchmark_results.csv` files. Can be run independently after `reproduce_all_runs.sh`. | ~5–30 min |

---

### Step 1 — First-time environment setup

`benchmark.py` automatically creates and manages its own virtual environment
(`.venv_benchmark/`). The first time you run any script, the setup happens
automatically. To force a reinstall:

**With GPU (NVIDIA CUDA — recommended if available):**
```bash
python3 benchmark/benchmark.py --force-setup --gpu
source .venv_benchmark/bin/activate
```

**CPU-only (no NVIDIA GPU):**
```bash
python3 benchmark/benchmark.py --force-setup --cpu-only
source .venv_benchmark/bin/activate
```

After setup, always activate the environment before running any script:
```bash
source .venv_benchmark/bin/activate
```

> **GPU vs CPU:** AnonShield v3.0 uses neural NER models (spaCy `en_core_web_lg`)
> that benefit from a GPU but work correctly on CPU. With `--cpu-only`, runs will
> be slower (roughly 2–5× for NER-heavy workloads) but results are identical.
> v1.0 and v2.0 are CPU-only regardless of this flag.

---

### Step 2 — Verify the pipeline with the smoke test (recommended first step)

Run this **before** the full reproduction to confirm your environment is
correctly configured. It uses tiny 500-row subsets of each dataset and completes
in ~5–20 min. At the end it also runs the scientific analysis on every result.

```bash
# GPU machine (default)
source .venv_benchmark/bin/activate
./paper_data/test_minimal/run_tests.sh

# CPU-only machine
source .venv_benchmark/bin/activate
./paper_data/test_minimal/run_tests.sh --cpu-only

# Dry run — print all commands without executing
./paper_data/test_minimal/run_tests.sh --dry-run
```

Expected output: `12/12` benchmark steps pass, `10/10` analysis steps pass,
`0` failures. CSV files are created under `paper_data/test_minimal/results/`.

> The `test_minimal/results/` directory is git-ignored and will be created fresh
> on each run. The smoke-test datasets are in `test_minimal/datasets/`.

---

### Step 3 — Full benchmark reproduction

Reproduces all runs from the paper using the complete datasets. Results are
written to `paper_data/results/`, matching the archived structure exactly.

> **Note on D2 (CAIS/CTCiber):** The D2 dataset is private and not included in
> this repository. `--skip-d2` (or the equivalent flag) will skip those runs.
> D3 (synthetic mock CVE) is public and fully reproducible.

```bash
source .venv_benchmark/bin/activate

# Full reproduction (~60–80 hours — all datasets)
./paper_data/scripts/reproduce_all_runs.sh

# CPU-only machine (slower, identical results)
./paper_data/scripts/reproduce_all_runs.sh --cpu-only

# Skip D1/D1C (fastest path — only D2/D3, ~30 h without-config runs)
./paper_data/scripts/reproduce_all_runs.sh --skip-d1

# Skip D2 (no access to private CAIS data)
./paper_data/scripts/reproduce_all_runs.sh --skip-d2

# D3 only (public dataset, ~3 h with-config, ~5 h without-config)
./paper_data/scripts/reproduce_all_runs.sh --skip-d1 --skip-d2

# Dry run — print all commands without executing
./paper_data/scripts/reproduce_all_runs.sh --dry-run
```

---

### Step 4 — Generate scientific analysis

After benchmarks complete, generate charts, statistical tables, and PDFs:

```bash
source .venv_benchmark/bin/activate

# Standard analyses (1–14, covers all paper figures and tables)
./paper_data/scripts/analyze_all.sh

# Extended analyses (1–17, includes additional exploratory plots)
./paper_data/scripts/analyze_all.sh --extended
```

Analysis output is written to an `analysis/` subfolder inside each result
directory (e.g., `paper_data/results/D3_mock_cve_csv__v3__all_strategies__10runs__with_config/analysis/`).

> The `analysis/` folders are git-ignored. They are regenerated each time
> `analyze_all.sh` runs.

### Versions and strategies reference

| Version | Description | Strategies accepted |
|---|---|---|
| v1.0 | Heuristic anonymizer, no OCR | *(built-in default, no flag needed)* |
| v2.0 | v1.0 + Tesseract OCR | *(built-in default, no flag needed)* |
| v3.0 | Neural NER + OCR + multiple engines | `filtered` `hybrid` `standalone` `presidio` |

> `--strategies default` is **not a valid CLI flag** — do not pass it. v1.0 and
> v2.0 use their internal default strategy automatically.
