# Anonymization Evaluation Guide

This document explains how to use the `src/anon/evaluation` module to measure the quality and effectiveness of the anonymization process. The pipeline is designed to provide quantitative metrics (Precision, Recall, F1-Score) by comparing the tool's output against a human-verified "ground truth."

## 1. Evaluation Philosophy

Effective evaluation is critical for ensuring that the anonymization tool correctly identifies and redacts Personally Identifiable Information (PII) without excessively redacting non-sensitive data. This module formalizes the evaluation process using a standard machine learning evaluation workflow.

- **True Positives (TP):** Sensitive entities that were correctly identified and anonymized.
- **False Positives (FP):** Non-sensitive text that was incorrectly identified and anonymized (over-anonymization).
- **False Negatives (FN):** Sensitive entities that were missed by the tool and left in the output (under-anonymization or PII leak).

The goal is to maximize Precision (minimize FP) and Recall (minimize FN).

## 2. The Evaluation Workflow

The process involves three main steps, orchestrated by the `EvaluationPipeline` class.

### Step 1: Prepare Data for Labeling

The first step is to generate a dataset for a human to label. The pipeline takes the output of the NER (Named Entity Recognition) generation mode (`--generate-ner-data`) and converts it into a format compatible with an annotation tool like [Doccano](https://github.com/doccano/doccano).

- **Input:** A `.jsonl` file where each line is a JSON object containing "text" and identified "label" annotations from the tool.
- **Action:** The `prepare_ground_truth` method in the `EvaluationPipeline` converts this file into a Doccano-compatible `.jsonl` file.
- **Key Class:** `evaluation.ground_truth.GroundTruthManager`

**Example Command:**
```bash
# First, generate the initial NER data
uv run anon.py /path/to/raw_documents/ --generate-ner-data --output-dir ner_output

# Then, prepare it for Doccano
uv run scripts/evaluate.py prepare \
    --ner-file ner_output/anon_raw_documents.jsonl \
    --doccano-export ground_truth_for_labeling.jsonl
```

### Step 2: Manual Labeling in Doccano

This is a manual step. The `ground_truth_for_labeling.jsonl` file must be imported into a Doccano project. A human reviewer then goes through each document to:
- Correct any wrong entity labels.
- Delete any incorrect entity spans (False Positives).
- Add any missing entity labels (False Negatives).

Once the review is complete, the corrected dataset must be exported from Doccano in the **JSONL** format. Let's call this file `ground_truth_corrected.jsonl`.

### Step 3: Generate Final Ground Truth

With the human-verified data, the next step is to create the final "gold standard" ground truth file. This file will contain the correct entities and their corresponding HMAC-SHA256 hashes, which are deterministically generated from the entity text and the `ANON_SECRET_KEY`.

- **Input:** The corrected `.jsonl` file from Doccano.
- **Action:** The `generate_ground_truth` method imports the Doccano data, generates the hashes for each entity, and saves the final ground truth to a single JSON file.
- **Key Class:** `evaluation.ground_truth.GroundTruth`

**Example Command:**
```bash
uv run scripts/evaluate.py generate \
    --doccano-corrected ground_truth_corrected.jsonl \
    --output-file ground_truth.json
```

### Step 4: Evaluate Anonymization Performance

The final step is to run the anonymizer on the original raw document and compare its output against the ground truth file generated in Step 3.

- **Inputs:**
    1. The anonymized output file (generated by running `anon.py` in standard mode).
    2. The `ground_truth.json` file.
- **Action:** The `evaluate_anonymization` method loads both files, compares the hashes found in the anonymized text against the expected hashes in the ground truth, and calculates the final metrics.
- **Key Classes:** `evaluation.metrics_calculator.MetricsCalculator`, `evaluation.hash_tracker.HashTracker`

**Example Command:**
```bash
# First, run anonymization on the original file
uv run anon.py /path/to/raw_documents/original_file.txt

# Then, evaluate the output
uv run scripts/evaluate.py evaluate \
    --anonymized-file output/anon_original_file.txt \
    --ground-truth-file ground_truth.json \
    --output-report evaluation_report.json
```
The console will print the Precision, Recall, and F1-Score, and a detailed JSON report will be saved.

## 3. Component Deep Dive

### `EvaluationPipeline` (`evaluation/__init__.py`)
This is the main orchestrator that ties all the steps together. It provides a clean, high-level interface for running the entire workflow.

### `GroundTruthManager` (`evaluation/ground_truth.py`)
- **Responsibility:** Manages the lifecycle of ground truth data.
- **Key Functions:**
    - `export_to_doccano()`: Converts NER output into a format suitable for labeling.
    - `import_from_doccano()`: Loads the human-corrected data.
    - `generate_ground_truth()`: Creates the final `GroundTruth` object by generating deterministic HMAC-SHA256 hashes for each entity, ensuring that the evaluation is consistent and secure.

### `GroundTruth` (`evaluation/ground_truth.py`)
- **Responsibility:** A data class that stores the final, authoritative ground truth.
- **Key Contents:**
    - A list of all `GroundTruthEntity` objects, including their text, type, start/end positions, and expected hashes.
    - The full original text of the documents being evaluated.

### `MetricsCalculator` (`evaluation/metrics_calculator.py`)
- **Responsibility:** Calculates the final performance metrics.
- **Key Functions:**
    - `calculate_metrics()`:
        1. Retrieves the set of expected hashes from the `GroundTruth` object.
        2. Uses `HashTracker` to find all anonymized hashes in the output text.
        3. Compares the two sets to calculate True Positives, False Positives, and False Negatives.
        4. Computes Precision, Recall, and F1-Score.

### `HashTracker` (`evaluation/hash_tracker.py`)
- **Responsibility:** A simple utility for finding and counting all anonymized slugs (e.g., `[PERSON_a1b2c3d4]`) in a block of text.
- **Key Functions:**
    - `count_hashes()`: Uses a regex to find all matching patterns and returns a count for each unique hash found.
